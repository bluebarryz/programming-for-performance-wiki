# Snoopy Caches

Snoopy caches are the simplest strategy for maintaining cache coherency in multiprocessor systems. The name comes from the fact that each cache "snoops" on (observes) the bus traffic generated by other caches, staying informed about what other CPUs are doing without relying on explicit notifications.

This is a distributed approach with no centralized state. Each cache controller monitors the shared bus. Whenever a CPU issues a memory write, the other CPUs observe the bus to see if that memory location exists in their cache. If so, they take action -- either invalidating their copy or updating it, depending on the protocol in use. All CPUs are connected to a shared bus, which is what makes snooping possible.

Snoopy caches form the foundation for all the cache coherency protocols discussed in the course: [[write-through-cache]], [[msi-protocol]], [[mesi-protocol]], and [[mesif-protocol]]. The shared-bus requirement is also a limitation -- snoopy protocols work well for systems with a moderate number of cores sharing a bus, but do not scale to very large systems where directory-based protocols may be needed instead.
